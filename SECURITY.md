# SECURITY.md

## Security Policy

Thank you for helping keep **SoftwareFactoryAI** projects safe.

This repository is **defensive-only**: it contains security checklists, playbooks, and validation guidance to help identify scams (e.g., “job interview trojans”) and reduce risk when reviewing third-party code.

---

## Reporting a Vulnerability / Security Concern

If you believe you found:
- a vulnerability in this repository,
- a security issue in the checklist/playbook content,
- or a real-world scam pattern you want to report responsibly,

please **do not open a public issue**.

### Preferred: Private report
Send a private report to:

- **Email:** braian@softwarefactoryai.com  

### What to include
- A short summary of the issue
- Impact / risk (what could happen)
- Reproduction steps (if applicable)
- Screenshots/logs (sanitized)
- Any indicators (domains, hashes, wallet addresses) **only if safe to share**
- Your suggested mitigation/fix (optional)

---

## Responsible Disclosure Guidelines

- **Do not publish malware, payloads, or weaponized proof-of-concepts** in this repo.
- **Do not share private data**, credentials, or personal information (yours or others).
- For suspicious repositories, share **sanitized evidence** and high-level indicators only.
- If you need a deeper technical review, share details **privately**.

---

## Scope

### In scope
- Security issues in repository content (misleading guidance, unsafe instructions)
- Vulnerabilities in any scripts/tools included in this repo (if any)
- Documentation that could unintentionally enable unsafe execution

### Out of scope
- Vulnerabilities in third-party code not maintained by SoftwareFactoryAI
- Requests to analyze or host malicious repositories publicly
- Incident response for third parties (we can share guidance, not act as IR for external orgs)

---

## Handling Reports

We aim to:
- acknowledge valid reports promptly,
- validate and assess impact,
- publish fixes/updates to the playbook when applicable,
- credit reporters when requested and appropriate.

---

## Safety Note

When in doubt: **assume compromise**, isolate the environment, and avoid running untrusted code outside a VM/sandbox.

— SoftwareFactoryAI
